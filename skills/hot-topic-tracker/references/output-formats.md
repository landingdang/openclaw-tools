# Output Formats

在需要结构化输出时，优先复用以下模板，而不是临时发散。

## 1. 歧义确认模板

```text
你输入的"[关键词]"可能指以下几个不同的对象：
1. [选项1] - [简要说明]
2. [选项2] - [简要说明]
3. [选项3] - [简要说明]
请确认你想追踪的是哪一个？
```

## 2. 无歧义时的结构化拆解

```text
- 主题：[明确的主题名称]
- 领域：[所属领域]
- 简介：[一句话说明这是什么]
- 用户可能关心的问题：
  1. [...]
  2. [...]
  3. [...]
- 建议搜索关键词：
  - [...]
  - [...]
  - [...]
- 追踪参数建议：
  - 时间范围：[如近 5 天]
  - 内容类型：[官方公告 / 新闻 / 评论 / 社交讨论]
  - 地域范围：[如全球 / 中国 / 美国]
```

## 3. 信源报告模板

```markdown
## 搜索主题：[主题名称]

### 信源概览
- 搜索时间范围：[时间范围]
- 有效信源数量：[数量]
- 核心发现：[一句话概括]

### 重要信源列表

**信源1：[标题]**
- 来源：[媒体/网站名称]
- 时间：[发布时间]
- 链接：[URL]
- 核心内容：[关键信息摘要]
- 重要数据/引语：[如有]
- 可信度：高/中/低

**信源2：[标题]**
- 来源：[媒体/网站名称]
- 时间：[发布时间]
- 链接：[URL]
- 核心内容：[关键信息摘要]
- 重要数据/引语：[如有]
- 可信度：高/中/低

### 信息交叉验证
- [重要事实1]：已被 [X] 个信源确认
- [重要事实2]：仅单一信源，需谨慎引用

### 搜索建议
- 信息缺口：[仍需补充的方向]
- 争议点：[不同信源存在分歧的内容]
```

## 4. 话题挖掘报告模板

```markdown
## 话题挖掘报告

### 分析过程
[简要说明从各信源中识别、合并、筛选话题的逻辑]

### 爆点话题列表

**话题1：[话题标题]**
- 事件发生时间：[YYYY-MM-DD]
- 时效性评级：[1天内 / 3天内 / 5天内]
- 与主题相关性：[为什么直接相关]
- 核心看点：[为什么值得写]
- 相关信源：[来源列表]

**话题2：[话题标题]**
- 事件发生时间：[YYYY-MM-DD]
- 时效性评级：[1天内 / 3天内 / 5天内]
- 与主题相关性：[为什么直接相关]
- 核心看点：[为什么值得写]
- 相关信源：[来源列表]

### 话题优先级建议
1. [最值得优先写的话题及原因]
2. [次优先话题及原因]
```

## 5. 文章模板

```markdown
# [20字以内标题]

[可选总述段。快速交代时间、地点、人物、事件、结果。]

## [有网感的章节标题]

正文内容，**关键数据**要加粗。

> "[人物原话]"——[姓名 / 机构 / 场合]

## [第二个章节标题]

正文内容。

## [第三个章节标题]

正文内容。
```

## 6. 事实核查报告模板

```markdown
## 事实核查报告

### 核查概览
- 文章标题：[标题]
- 抽取陈述数：[X]条
- 核查结果：✅ [X]条确认 / ⚠️ [X]条存疑 / ❌ [X]条有误

### 核查详情

**陈述1：**[具体陈述内容]
- 核查结果：✅ 已确认
- 验证来源：[信源名称 + 链接]

**陈述2：**[具体陈述内容]
- 核查结果：⚠️ 存疑
- 问题说明：[为什么无法确认或信源冲突]
- 处理建议：[保留但降级表述 / 删除 / 补充限定词]

**陈述3：**[具体陈述内容]
- 核查结果：❌ 有误
- 问题说明：[具体问题]
- 正确信息：[正确内容]
- 修正建议：将"..."改为"..."

### 修正清单
- [修正项1]
- [修正项2]

### 核查结论
[总体评价文章的事实准确性]
```

## 7. 常见失误检查清单

- 没先搜原始关键词就直接判断主题
- 歧义存在却没有暂停确认
- 话题与原主题只有弱关联
- 把发布时间误当作事件发生时间
- 纳入超过 5 天的旧闻但没有说明它仍在演进
- 只依赖单一信源
- 文章里出现未经核实的数据或引语
- 写完就交付，没有做事实核查
