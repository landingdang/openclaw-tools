# OpenClaw 智能体模板

这个目录用于覆盖智能体 workspace 的默认定义文件。

## 配置文件关系图

```
启动流程：
┌─────────────────┐
│ BOOTSTRAP.md    │ 首次启动时执行，完成后自动删除
└────────┬────────┘
         ↓
┌─────────────────┐
│ AGENTS.md       │ 定义加载顺序和全局契约
└────────┬────────┘
         ↓
    加载顺序：
    ┌──────────────┐
    │ IDENTITY.md  │ 身份卡与系统字段
    └──────┬───────┘
           ↓
    ┌──────────────┐
    │ SOUL.md      │ 表达风格、判断气质
    └──────┬───────┘
           ↓
    ┌──────────────┐
    │ USER.md      │ 主人画像与长期偏好
    └──────┬───────┘
           ↓
    ┌──────────────┐
    │ TOOLS.md     │ 操作边界、SKILL 路由（路由权威来源）
    └──────┬───────┘
           ↓
    ┌──────────────────┐
    │ TASK_TRACKING.md │ 任务追踪机制（强制 TODO）
    └──────────────────┘

运行时：
┌─────────────────┐
│ HEARTBEAT.md    │ 定时任务（每天 08:00 或手动触发）
└─────────────────┘

┌─────────────────┐
│ LONG_TASK.md    │ 长时任务执行规范（预计耗时 > 2 分钟时读取）
└─────────────────┘

任务执行流程：
用户任务 → 判断是否需要 TODO → 创建 TODO 列表 → 逐项执行 → 标记完成 → 验证产物
           ↑                                    ↑
           └─── TASK_TRACKING.md ───────────────┘
```

## 文件职责

| 文件 | 职责 | 何时读取 | 可否修改 |
|---|---|---|---|
| `BOOTSTRAP.md` | 首次启动配置采集流程 | 仅首次启动 | 完成后删除 |
| `AGENTS.md` | 加载顺序、全局契约、SKILL 路由原则 | 每次会话开始 | 可修改 |
| `IDENTITY.md` | 身份卡、系统字段、占位符说明 | 每次会话开始 | 部分可修改 |
| `SOUL.md` | 回复风格、语言规范、禁止行为 | 每次会话开始 | 可修改 |
| `USER.md` | 用户画像、偏好、历史反馈 | 每次会话开始 | 可修改 |
| `TOOLS.md` | 操作边界、SKILL 路由决策树 | 每次会话开始 | 可修改 |
| `TASK_TRACKING.md` | 任务追踪机制、TODO 创建与验证 | 每次会话开始 | 可修改 |
| `HEARTBEAT.md` | 定时任务执行流程 | 定时或手动触发 | 可修改 |
| `LONG_TASK.md` | 长时任务执行规范、进度汇报 | 执行长时任务时 | 可修改 |

## 核心设计原则

### 1. 职责分离
- **TOOLS.md** 是 SKILL 路由的唯一权威来源，其他文件不重复维护路由表
- **SOUL.md** 只管表达风格，不管业务逻辑
- **USER.md** 只记录用户偏好，不记录系统配置

### 2. 可执行性优先
- 所有规则都可量化验证（字数限制、禁止词表、决策树）
- 避免主观描述（"话少""克制""可信"）
- 优先使用"必须/禁止"而非"尽量/避免"

### 3. 大模型友好
- 决策树 > 自然语言描述
- 强制路由关键词 > 意图理解
- 明确的默认行为 > 开放式判断

## 覆盖规则

- 同名文件优先使用 `template/` 版本
- 没有模板时回退到脚本内置默认内容
- `IDENTITY.md` 这类需要系统字段的文件，脚本会在模板基础上自动补齐必要元数据
- 项目根目录的 `skills/` 会同步复制到 OpenClaw 根目录下的共享 `skills/`，供所有智能体共用

|---|---|---|
| `{{agent_id}}` | 智能体唯一标识 | `openclaw-001` |

## 快速排查指南

### 问题：智能体回复太啰嗦
**检查文件：** `SOUL.md` → 回复长度控制  
**检查文件：** `USER.md` → 回复长度偏好  
**可能原因：** 未设置字数限制或用户偏好为"详细"

### 问题：智能体没有调用正确的 skill
**检查文件：** `TOOLS.md` → SKILL 路由决策树  
**检查文件：** `USER.md` → 高频任务偏好  
**可能原因：** 
1. 关键词未在强制路由表中
2. 用户偏好覆盖了默认路由
3. 输入类型识别错误

### 问题：智能体问太多问题，不直接执行
**检查文件：** `SOUL.md` → 禁止的"空问题"  
**检查文件：** `USER.md` → 是否接受先执行后汇报  
**可能原因：** 决策偏好设置为"稳妥"

### 问题：智能体在群聊里乱发言
**检查文件：** `TOOLS.md` → 对外动作规则  
**检查文件：** `AGENTS.md` → 全局契约（群聊规则）  
**可能原因：** 未正确识别消息归属或群聊场景

### 问题：心跳任务没有按时执行
**检查文件：** `HEARTBEAT.md` → 执行时机  
**检查系统：** 定时任务配置（cron 或系统调度）  
**可能原因：** 定时任务未正确配置

### 问题：智能体执行任务时长时间无响应
**检查文件：** `LONG_TASK.md` → 里程碑汇报规则  
**检查文件：** `TASK_TRACKING.md` → TODO 创建规则  
**检查文件：** 对应 skill 的 SKILL.md → 是否明确了步骤  
**可能原因：** 
1. 任务未识别为长时任务
2. 未创建 TODO 列表
3. Skill 未实现里程碑汇报
4. 网络请求超时

### 问题：智能体说完成了但没有产物
**检查文件：** `TASK_TRACKING.md` → 产物验证规则  
**检查文件：** `AGENTS.md` → 全局契约（禁止假装完成）  
**可能原因：** 
1. 未创建 TODO，跳过了验证步骤
2. TODO 标记为完成但未提供证据
3. 产物路径错误

### 问题：首次启动配置不完整
**检查文件：** `BOOTSTRAP.md` → 执行流程  
**检查文件：** `IDENTITY.md` / `USER.md` → 必填字段  
**可能原因：** 
1. BOOTSTRAP.md 被提前删除
2. 用户跳过了某些配置步骤

## 配置修改命令

```bash
# 修改智能体名字
/config set name 新名字

# 修改签名表情
/config set signature_emoji 🦾

# 修改用户称呼
/config set user_name 老板

# 修改决策风格
/config set decision_style 快速

# 修改回复长度偏好
/config set reply_style 简短

# 查看当前配置
/config show

# 重置配置（重新触发 BOOTSTRAP）
/config reset
```

## 最佳实践

### 1. 首次配置
- 完整走完 BOOTSTRAP 流程，不要跳过
- 明确告知高频任务类型，让智能体记录偏好
- 设置清晰的沟通禁忌（如不要用某些表达）

### 2. 日常使用
- 发现智能体行为不符合预期时，先检查 USER.md 是否有冲突偏好
- 新增 skill 后，及时更新 TOOLS.md 的路由表
- 定期查看 USER.md 的历史反馈记录，清理过时条目

### 3. 多智能体管理
- 通用规则放在 template/ 目录
- 个性化配置放在各智能体的 workspace
- skill 定义统一放在共享 skills/ 目录

### 4. 任务执行最佳实践
- 所有非即时任务都创建 TODO
- 每个 TODO 完成后立即标记
- 每个 TODO 提供证据（路径、尺寸、耗时）
- 任务完成后验证所有产物

### 5. 调试技巧
- 智能体行为异常时，先问"你读取了哪些配置文件？"
- 怀疑路由错误时，问"为什么选择了这个 skill？"
- 需要验证配置时，问"USER.md 里记录了什么？"
- 怀疑假装完成时，问"显示 TODO 列表和完成证据"

## 示例场景

### 场景 1：配置一个简洁高效的技术助手
```markdown
# IDENTITY.md
- 名字：TechBot
- 身份：技术顾问
- 签名表情：⚡

# USER.md
- 称呼：老板
- 决策风格：快速
- 回复长度：简短
- 技术深度：专家
- 高频任务：代码审查、技术文档翻译
```

### 场景 2：配置一个温暖的生活助手
```markdown
# IDENTITY.md
- 名字：小助手
- 身份：数字伙伴
- 签名表情：🤗

# USER.md
- 称呼：主人
- 决策风格：稳妥
- 回复长度：适中
- 高频任务：日程管理、消息整理、热点追踪
```
