# LONG_TASK.md

## 长时任务定义

满足以下任一条件的任务视为"长时任务"：

- **预计执行时间 > 2 分钟**
- **需要多步骤串行执行**（3 步及以上）
- **涉及外部 API 调用**（网络搜索、图片生成、文档处理）
- **需要等待外部资源**（文件下载、模型推理）
- **批量操作**（多文件处理、批量生成）

## 强制执行规则

### 1. 任务开始前：预告机制

**必须在任务开始前输出预告信息：**

```
开始执行：[任务名称]

预计耗时：[X 分钟]

执行步骤：
1. [步骤 1 名称] (预计 X 秒)
2. [步骤 2 名称] (预计 X 秒)
3. [步骤 3 名称] (预计 X 秒)

重要里程碑：
• [里程碑 1]
• [里程碑 2]

请稍候，我会在关键节点向你汇报进度。
```

**预计耗时计算规则：**

| 操作类型 | 预计耗时 |
|---|---|
| 网络搜索（单次） | 5-10 秒 |
| 网页抓取（单页） | 10-30 秒 |
| 文章翻译（1000 字） | 30-60 秒 |
| 图片生成（单张） | 20-40 秒 |
| 图片生成（批量 5 张） | 2-3 分钟 |
| 文档生成（DOCX/PDF） | 30-60 秒 |
| PPT 生成（10 页） | 2-3 分钟 |
| 热点追踪（完整流程） | 3-5 分钟 |
| 文章配图（5 张） | 3-4 分钟 |

### 2. 执行过程中：里程碑汇报

**每完成一个重要里程碑，必须立即输出进度：**

```
✓ 已完成：[里程碑名称]
  结果：[简短描述，1 句话]
  
进行中：[下一步骤名称]
```

**里程碑识别规则：**

- 完成外部 API 调用（搜索完成、图片生成完成）
- 完成文件读写（文件保存成功）
- 完成关键决策（确认配图方案、选择模板）
- 完成阶段性产物（大纲生成、prompt 文件写入）

**禁止行为：**
- ❌ 连续执行多个步骤后才汇报
- ❌ 只在任务结束时才输出结果
- ❌ 使用"正在处理..."后长时间无响应

### 3. 任务结束后：结果摘要

**任务完成时输出简洁摘要：**

```
任务完成：[任务名称]

实际耗时：[X 分钟 Y 秒]

产出：
• [产物 1]：[路径或描述]
• [产物 2]：[路径或描述]

[可选：下一步建议]
```

## 任务分拆规则

### 何时必须分拆

- **预计耗时 > 5 分钟** → 必须分拆为多个子任务，逐个执行
- **步骤数 > 5** → 按逻辑分组，分批执行
- **批量操作 > 10 项** → 分批处理，每批汇报一次

### 分拆示例

**原任务：** "抓取 10 篇文章并翻译"

**分拆后：**
```
任务 1：抓取 10 篇文章（预计 3 分钟）
  → 完成后汇报：已保存 10 篇文章到 media/articles/

任务 2：翻译文章 1-5（预计 3 分钟）
  → 完成后汇报：已完成前 5 篇翻译

任务 3：翻译文章 6-10（预计 3 分钟）
  → 完成后汇报：全部翻译完成
```

## 用户等待体验优化

### 禁止的"静默执行"

❌ **错误示例：**
```
用户：帮我翻译这篇文章
智能体：好的
[静默 2 分钟]
智能体：翻译完成，保存在 translation.md
```

✅ **正确示例：**
```
用户：帮我翻译这篇文章
智能体：
开始翻译（refined 模式）
预计耗时：2-3 分钟

执行步骤：
1. 分析文章结构和术语
2. 生成翻译 prompt
3. 生成初稿
4. 批评审阅
5. 修订润色

✓ 分析完成 → 识别 15 个专业术语
✓ 初稿完成 → 03-draft.md
✓ 批评审阅完成 → 发现 3 处需优化
✓ 最终润色完成

任务完成：翻译
实际耗时：2 分 18 秒
产出：translation.md
```

### 超长任务的中断机制

**预计耗时 > 10 分钟时，必须：**

1. 在预告中明确说明总耗时
2. 询问用户是否继续
3. 提供分批执行方案

**示例：**
```
这个任务预计需要 15 分钟（翻译 20 篇文章）

建议方案：
A. 一次性执行（15 分钟）
B. 分 3 批执行（每批 5 分钟，可随时中断）
C. 先执行前 5 篇（3 分钟），确认效果后继续

请选择方案，或告诉我你的偏好。
```

```

• 已完成：[里程碑列表]

是否继续？
A. 继续等待
B. 中断并保存当前进度
C. 取消任务
```

## 检查清单

在执行长时任务前，确认：

- [ ] 已输出任务预告（包含预计耗时和步骤）
- [ ] 已识别所有重要里程碑
- [ ] 预计耗时 > 5 分钟时，已询问用户是否分拆
- [ ] 每个里程碑完成后立即汇报
- [ ] 任务完成后输出结果摘要
- [ ] 异常情况下给出明确的失败原因和建议

## 与 USER.md 的关系

如果 `USER.md` 中设置了：
- **决策风格 = 快速** → 可以减少确认环节，但里程碑汇报仍然必须
- **是否接受先执行后汇报 = 是** → 可以直接开始，但预告和里程碑汇报仍然必须
- **回复长度 = 简短** → 里程碑汇报可以更简洁，但不能省略

**核心原则：长时任务的进度透明度不受用户偏好影响，始终保持。**
