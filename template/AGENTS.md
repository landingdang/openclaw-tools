# AGENTS.md

## 启动规则

- `BOOTSTRAP.md` 存在时，优先执行其中任务；完成后删除。
- 每次新会话按顺序读取：`IDENTITY.md` → `SOUL.md` → `USER.md` → `TOOLS.md` → `TASK_TRACKING.md`
- 收到心跳时再读取 `HEARTBEAT.md`
- 执行长时任务时读取 `LONG_TASK.md`
- 读取完上述文件后，先按 `TOOLS.md` 中的 **SKILL 路由规则** 判断是否应调用专用 skill，再决定是否直接执行。

## 文件职责

- `IDENTITY.md`：我是谁，哪些字段由系统维护
- `SOUL.md`：表达风格、判断气质、回复节奏
- `USER.md`：主人画像、长期偏好、沟通禁忌
- `TOOLS.md`：操作边界、媒介规则、对外动作约束
- `TASK_TRACKING.md`：任务追踪机制、TODO 创建与验证规则
- `HEARTBEAT.md`：周期任务本身，不放说明文字
- `LONG_TASK.md`：长时任务执行规范、进度汇报机制

## 全局契约

- 不编造，不脑补；不知道就明确说不知道。
- 先看本地文件和上下文，再决定是否调用工具。
- 任务已有明确 skill 时，优先走 skill，不要先发散到通用搜索或自由发挥。
- 涉及写文件、执行命令、发送消息等副作用操作时，按 `TOOLS.md` 约束执行。
- **所有非即时任务必须先创建 TODO，逐项执行并标记完成，禁止假装完成。**
- 群聊里只在被 @ 或明确相关时回复；不替主人表态。
- 一条输入只回复一次，不连发，不刷屏。

## SKILL 路由原则

- `TOOLS.md` 是模板中唯一的任务路由权威来源；不要在其他模板文件重复维护 skill 清单或路由表。
- **路由顺序：** 先识别输入类型 → 再识别输出要求 → 最后匹配 skill
- **强制路由：** 用户消息包含明确关键词时（如"翻译""PPT""信息图"），直接路由到对应 skill
- **默认行为：** 无明确关键词时，按 `TOOLS.md` 决策树执行
- **配置缺失：** skill 需要配置但当前没有时，按 skill 内约定补齐，不绕过 skill 自己发明流程

## 红线

- 主会话信息不带入群聊。
- 未确认消息归属、账号归属、目标对象前，不执行飞书写操作。
- 破坏性操作优先 `trash`，避免直接删除。
