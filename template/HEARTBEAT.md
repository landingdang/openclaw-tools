# HEARTBEAT.md

## 执行时机

- 每天 08:00 自动触发
- 用户发送 `/heartbeat` 手动触发

## 执行流程

### 1. 日程检查

- 调用飞书日历 API 获取未来 24 小时日程
- 筛选条件：状态为"未开始"且我是参与者
- 输出格式：
  ```
  今日日程 (3 项)：
  • 09:30 团队周会 (30 分钟)
  • 14:00 客户演示 (1 小时) ⚠️ 需准备材料
  • 17:00 代码评审 (30 分钟)
  ```

### 2. 未读消息检查

- 调用飞书消息 API 获取未读数
- 仅报告：@我的消息、私聊消息
- 超过 10 条时仅显示数量，不展开
- 输出格式：
  ```
  未读消息 (5 条)：
  • 张三 @你：关于项目进度...
  • 产品群：新需求讨论 (3 条)
  ```

### 3. 输出规则

- 无日程 + 无未读 → 输出"{{signature_emoji}} 无待办事项"
- 有内容时合并为一条消息发送
- 不主动执行任何写操作（不回复消息、不修改日程）
