# TASK_TRACKING.md

## 任务追踪机制

**核心理念：** 所有非即时任务都必须先创建 TODO，逐项完成并标记，禁止"假装完成"。

---

## 强制使用 TODO 的场景

### 必须创建 TODO 的任务类型

| 任务类型 | 判断标准 | 示例 |
|---|---|---|
| **长时任务** | 预计耗时 > 2 分钟 | 翻译长文、批量生成图片、热点追踪 |
| **多步骤任务** | 步骤数 ≥ 3 | 网页抓取 → 翻译 → 配图 |
| **需要外部调用** | 调用 skill、API、命令行 | 搜索、出图、文档生成 |
| **批量操作** | 处理数量 > 1 | 翻译 5 篇文章、生成 10 张图 |
| **用户明确要求** | 用户说"帮我做 X、Y、Z" | 列表式任务 |

### 可以不创建 TODO 的场景

- 单步即时查询（读文件、简单问答）
- 纯分析任务（代码审查、文本分析）
- 配置修改（更新 USER.md）

## TODO 创建规则

### 1. 任务拆解原则

**SMART 原则：**
- **Specific（具体）：** "翻译文章" → "翻译 article.md（3000 字，refined 模式）"
- **Measurable（可衡量）：** "生成图片" → "生成 5 张产品图（1024x1024）"
- **Achievable（可完成）：** 单个 TODO 不超过 5 分钟
- **Relevant（相关）：** 每个 TODO 都是最终目标的必要步骤
- **Time-bound（有时限）：** 标注预计耗时

**拆解示例：**

❌ **错误拆解：**
```
TODO: 完成文章翻译和配图
```

✅ **正确拆解：**
```
TODO 1: 读取文章内容并分析结构（预计 30 秒）
TODO 2: 翻译文章（refined 模式，预计 3 分钟）
TODO 3: 分析文章并规划配图位置（预计 1 分钟）
TODO 4: 生成 5 张插图（预计 3 分钟）
TODO 5: 将插图插入文章（预计 30 秒）
```

### 2. TODO 命名规范

**格式：** `[动词] + [对象] + [关键参数]`

**示例：**
- ✅ "翻译 article.md（3000 字，refined 模式）"
- ✅ "生成封面图（主题：AI 技术，风格：商务简约）"
- ✅ "搜索信源（关键词：人工智能，时间范围：近 7 天）"
- ❌ "处理文章"（太模糊）
- ❌ "做一些准备工作"（无法验证）

### 3. TODO 依赖关系

**串行依赖：** 使用 `blockedBy` 标记

```
TODO 1: 抓取网页内容
TODO 2: 翻译网页内容（blockedBy: TODO 1）
TODO 3: 生成封面图（blockedBy: TODO 2）
```

**并行任务：** 不设置 `blockedBy`

```
TODO 1: 翻译文章 A
TODO 2: 翻译文章 B
TODO 3: 翻译文章 C
（可并行执行）
```

## TODO 执行流程

### 阶段 1：任务接收与拆解

**用户输入任务后，立即：**

1. **判断是否需要创建 TODO**
   - 符合"必须创建 TODO"的场景 → 继续
   - 不符合 → 直接执行

2. **拆解任务并创建 TODO 列表**
   ```
   收到任务：[任务描述]
   
   任务拆解：
   1. [步骤 1]（预计 X 秒）
   2. [步骤 2]（预计 X 秒）
   3. [步骤 3]（预计 X 秒）
   
   总预计耗时：[X 分钟]
   
   创建 TODO 列表...
   ```

3. **调用 TaskCreate 创建所有 TODO**
   - 每个 TODO 包含：subject、description、预计耗时
   - 设置依赖关系（blockedBy）
   - 状态初始为 `pending`

4. **输出 TODO 列表供用户确认**
   ```
   已创建任务列表：
   
   □ TODO 1: [任务名称]（预计 X 秒）
   □ TODO 2: [任务名称]（预计 X 秒）
   □ TODO 3: [任务名称]（预计 X 秒）
   
   总预计耗时：[X 分钟]
   
   开始执行...
   ```

### 阶段 2：逐项执行

**执行规则：**

1. **开始执行前：** 调用 `TaskUpdate` 标记为 `in_progress`
   ```
   ▶ 开始执行：TODO 1 - [任务名称]
   ```

2. **执行任务：** 调用对应的 skill、工具、命令
   - 必须真实执行，不能跳过
   - 必须等待执行完成，不能假装完成

3. **执行完成后：** 立即调用 `TaskUpdate` 标记为 `completed`
   ```
   ✓ 已完成：TODO 1 - [任务名称]
     结果：[简短描述]
     实际耗时：[X 秒]
   ```

4. **执行失败时：** 保持 `in_progress`，输出错误信息
   ```
   ✗ 失败：TODO 1 - [任务名称]
     错误：[错误信息]
     
   是否继续？
   A. 重试
   B. 跳过此步骤
   C. 终止任务
   ```

### 阶段 3：任务完成

**所有 TODO 完成后：**

1. **调用 TaskList 验证所有任务状态**
   ```bash
   # 验证命令
   所有任务状态：
   ✓ TODO 1: completed
   ✓ TODO 2: completed
   ✓ TODO 3: completed
   ```

2. **输出最终摘要**
   ```
   任务完成：[任务名称]
   
   实际耗时：[X 分钟 Y 秒]
   预计耗时：[X 分钟]
   
   完成情况：
   ✓ TODO 1: [任务名称]
   ✓ TODO 2: [任务名称]
   ✓ TODO 3: [任务名称]
   
   产出：
   • [产物 1]
   • [产物 2]
   ```

## 防止"假装完成"的机制

### 1. 强制验证点

**每个 TODO 完成时必须提供证据：**

| TODO 类型 | 必须提供的证据 |
|---|---|
| 文件读取 | 文件路径 + 内容摘要（前 100 字） |
| 文件写入 | 文件路径 + 文件大小 |
| 网络搜索 | 搜索关键词 + 结果数量 + 前 3 条标题 |
| 图片生成 | 图片路径 + 尺寸 + provider/model |
| 文档生成 | 文档路径 + 页数/字数 |
| Skill 调用 | Skill 名称 + 输入参数 + 输出路径 |
| 命令执行 | 命令 + 退出码 + 关键输出 |

**示例：**

❌ **假装完成：**
```
✓ 已完成：翻译文章
  结果：翻译完成
```

✅ **真实完成：**
```
✓ 已完成：翻译文章
  结果：translation.md（1247 字，refined 模式）
  路径：media/baoyu-translate/article-zh/translation.md
  实际耗时：2 分 18 秒
```

### 2. 时间验证

**记录每个 TODO 的开始和结束时间：**

```
▶ 开始执行：TODO 1（10:23:15）
✓ 已完成：TODO 1（10:23:47）
  实际耗时：32 秒
  预计耗时：30 秒
```

**异常检测：**
- 实际耗时 < 预计耗时的 20% → 可能假装完成
- 实际耗时 > 预计耗时的 300% → 可能卡住

### 3. 依赖验证

**执行 TODO 前检查依赖：**

```
▶ 准备执行：TODO 2 - 翻译网页内容
  依赖检查：
  ✓ TODO 1（抓取网页内容）已完成
  ✓ 输入文件存在：media/baoyu-url-to-markdown/article.md
  
开始执行...
```

**依赖未满足时禁止执行：**

```
✗ 无法执行：TODO 2 - 翻译网页内容
  原因：TODO 1（抓取网页内容）未完成
  
请先完成 TODO 1
```

### 4. 产物验证

**任务完成后验证产物是否存在：**

```
任务完成验证：
✓ 检查文件：translation.md（存在，1247 字）
✓ 检查文件：01-analysis.md（存在，523 字）
✓ 检查文件：02-prompt.md（存在，312 字）

所有产物验证通过
```

## 与现有机制的集成

### 1. 与 LONG_TASK.md 的关系

**LONG_TASK.md：** 定义长时任务的预告、里程碑汇报规范  
**TASK_TRACKING.md：** 定义 TODO 创建、执行、验证机制

**协同工作：**
```
用户：翻译这篇文章（refined 模式）

智能体：
1. 判断：这是长时任务（预计 3 分钟）
2. 读取 LONG_TASK.md → 需要预告和里程碑汇报
3. 读取 TASK_TRACKING.md → 需要创建 TODO
4. 创建 TODO 列表（5 个步骤）
5. 输出预告（包含 TODO 列表）
6. 逐项执行 TODO，每完成一个汇报里程碑
7. 全部完成后输出摘要
```

### 2. 与 TOOLS.md 的关系

**TOOLS.md：** 定义 skill 路由规则  
**TASK_TRACKING.md：** 定义任务执行追踪

**协同工作：**
```
用户：翻译这篇文章

智能体：
1. 读取 TOOLS.md → 路由到 baoyu-translate
2. 读取 TASK_TRACKING.md → 创建 TODO
3. TODO 1: 读取配置文件
4. TODO 2: 调用 baoyu-translate skill
5. TODO 3: 验证输出文件
```

### 3. 与 USER.md 的关系

**USER.md 中的决策风格影响 TODO 粒度：**

| 决策风格 | TODO 粒度 | 汇报频率 |
|---|---|---|
| 稳妥 | 细粒度（每个小步骤） | 每个 TODO 完成后汇报 |
| 快速 | 中粒度（逻辑分组） | 每组 TODO 完成后汇报 |
| 激进 | 粗粒度（大步骤） | 仅关键里程碑汇报 |

**示例：**

**稳妥风格：**
```
TODO 1: 读取文章文件
TODO 2: 分析文章结构
TODO 3: 提取术语表
TODO 4: 生成翻译 prompt
TODO 5: 调用翻译 API
TODO 6: 保存翻译结果
TODO 7: 验证输出文件
```

**快速风格：**
```
TODO 1: 准备翻译（读取文件 + 分析结构）
TODO 2: 执行翻译（生成 prompt + 调用 API）
TODO 3: 保存结果（保存文件 + 验证）
```

**激进风格：**
```
TODO 1: 翻译文章（完整流程）
```

## 检查清单

在执行任务前，确认：

- [ ] 已判断是否需要创建 TODO
- [ ] 已按 SMART 原则拆解任务
- [ ] 已创建所有 TODO（调用 TaskCreate）
- [ ] 已设置依赖关系（blockedBy）
- [ ] 已输出 TODO 列表供用户确认
- [ ] 每个 TODO 开始前标记为 in_progress
- [ ] 每个 TODO 完成后标记为 completed
- [ ] 每个 TODO 完成后提供证据（路径、尺寸、耗时）
- [ ] 所有 TODO 完成后验证产物
- [ ] 输出最终摘要

## 总结

**核心价值：**
1. **防止假装完成：** 每个 TODO 必须提供证据
2. **提升透明度：** 用户随时知道进度
3. **便于调试：** 任务失败时能快速定位
4. **强制执行：** TODO 机制比纯文档约束更强

**关键机制：**
- 强制创建 TODO（非即时任务）
- 逐项执行并标记状态
- 每个 TODO 提供证据
- 验证依赖和产物
- 记录时间和耗时

**预期效果：**
- 智能体无法"偷懒"跳过步骤
- 用户体验显著提升
- 任务执行更可靠
- 问题排查更容易
